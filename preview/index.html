Webflow Animation Scripts

Drop-in GSAP animations controlled via data attributes. One script, all animations. Just add attributes in Webflow.

Quick Setup

Add this single script tag to your Webflow page or site-wide custom code (before </body>):

<script src="https://webflow-scripts-two.vercel.app/index.js"></script>

Then add data-* attributes to your Webflow elements. All animations below are included in this one file.

For local development: <script src="http://localhost:3333/dist/index.js"></script>

Start dev server: cd webflow-scripts && npm run dev

parallax scroll-driven

Two modes: per-element parallax within a section, and background-image parallax.

Section parallax (elements move at different rates)

<div data-parallax>
  <div data-parallax-speed="0.2">Slow element</div>
  <div data-parallax-speed="0.5">Medium element</div>
  <div data-parallax-speed="1.0">Fast element</div>
</div>
AttributeWhereWhat it does
data-parallax Section/wrapper Marks the container as a parallax section
data-parallax-speed="0.5" Child elements How much the element moves. 0 = static, 0.5 = subtle, 1+ = strong, negative = reverse

Background parallax

<div data-parallax-bg data-parallax-speed="0.3" style="background-image: url(…)">
  Content over parallax background
</div>
AttributeWhereWhat it does
data-parallax-bg Element with CSS background image Enables background-position parallax
data-parallax-speed="0.3" Same element Controls parallax intensity (default 0.3)

hero-animation scroll-triggered

Fade-in + slide-up animation for elements as they enter the viewport.

Usage

<div data-animate="fade-up">This will fade in and slide up</div>
AttributeWhereWhat it does
data-animate="fade-up" Any element Element fades in and slides up 60px when it enters the viewport

horizontal-scroll pinned

Pins a section and scrolls its content horizontally as the user scrolls vertically.

Structure

<section data-horizontal-scroll>
  <div data-horizontal-track>
    <div>Card 1</div>
    <div>Card 2</div>
    <div>Card 3</div>
  </div>
</section>

Attributes

Required CSS (100vh, overflow hidden, flex) is applied automatically by the script. Just give cards a fixed width in Webflow.

AttributeWhereWhat it does
data-horizontal-scroll Section wrapper Marks section for pin + horizontal scroll
data-horizontal-track Inner flex container The element that moves horizontally
data-horizontal-speed="1" Section (optional) Scroll multiplier. 0.5 = faster, 2 = slower/longer

stagger-up scroll-triggered

Children of a container stagger up from below with rotation and x offset.

Usage

<div data-stagger-up>
  <div>Child 1</div>
  <div>Child 2</div>
  <div>Child 3</div>
</div>

Attributes (all on the parent container)

AttributeDefaultWhat it does
data-stagger-up -- Enables the stagger animation on direct children
data-stagger-delay="0.15" 0.15 Delay in seconds between each child
data-stagger-duration="0.8" 0.8 Animation duration per child
data-stagger-rotation="6" 6 Starting rotation in degrees
data-stagger-x="30" 30 Starting horizontal offset in px

The parent gets overflow: hidden automatically so children are clipped during animation.

Development

Source: github.com/Koalitionist/webflow-animations

Local dev

npm run dev      # Start Vite dev server on port 3333
npm run build    # Build dist/index.js with esbuild

Adding a new animation

Add a new IIFE section in scripts/index.js. Use the bundled gsap and ScrollTrigger variables directly (they're in scope). Query elements by data-* attributes, create animations, and wrap in an element-existence check so it's a no-op on pages without those elements.

Architecture

Single index.js bundle built with esbuild. Bundles its own GSAP 3.14.2 + ScrollTrigger to avoid conflicts with Webflow's mismatched GSAP globals. Webflow's window.gsap is temporarily hidden during bundle initialization, then restored.
